핵심 요구사항 분석

  • MVP 고도화 — 동작하는 MVP를 기반으로 화면 기획을 고도화하고 상용화 수준의 시각적 완성도 확보 (7~8월 양대 마켓 정식 런칭 목표)
  • iOS·Android 호환 설계 — 양대 OS 환경에 맞는 화면 기획 및 UI/UX 디자인, 플랫폼별 UI 가이드라인(HIG / Material) 준수
  • 사용자 계정 및 권한 모듈 — 일반 유저 소셜 로그인, 제휴 헬스장 점주 가입·인증, 유저 유형별 마이페이지/프로필
  • 이용권 관리 모듈 — 보유 이용권·PT 횟수 등록, 이용권 상태 조회 및 일정 관리 화면
  • 검색 및 결제 모듈 — 위치 기반/지점별 헬스장 검색, 센터 상세, 이용권 결제 연동, 결제 완료 후 모바일 출입증 발급
  • 제휴 헬스장(점주) 관리 모듈 — 방문 예정자 확인·알림, 사용자 출입 승인 처리, 이용 내역 통계 및 정산 대시보드
  • 개발 협업 및 산출물 이관 — 내부 풀스택 개발자와의 원활한 소통과 즉시 구현 가능한 산출물(설계서·정의서·Figma) 전달

제안 포인트

1. 피트니스 O2O 도메인을 이미 끝까지 만들어 본 경험. 회원·강사·센터관리자 멀티롤, 수업/이용권 예약, 결제, 매출 대시보드를 갖춘 피트니스 프랜차이즈 O2O 플랫폼을 기획·디자인·개발 전 범위로 수행했습니다. 본 프로젝트의 일반 유저 / 점주 이중 구조, 이용권 결제, 정산 대시보드 요구사항을 도메인 맥락 그대로 이해하고 설계합니다.

2. 개발자가 바로 구현하는 설계서 품질. 화면 기획을 단순 와이어프레임에 그치지 않고, 상태·예외·분기·데이터 항목까지 명시한 기능 정의서로 산출합니다. 내부 풀스택 개발자가 추가 질의 없이 구현에 착수할 수 있도록 핸드오프 단계에서 컴포넌트 스펙·인터랙션 명세·에셋을 함께 전달합니다.

3. 출입증·결제 등 상태 흐름이 복잡한 화면에 강점. 결제 완료 → 모바일 출입증 발급 → 점주 출입 승인으로 이어지는 상태 전이, 이용권 잔여 횟수·만기 추적 등은 화면이 아니라 상태 설계가 핵심입니다. 다수 모바일 앱에서 결제·예약·알림의 상태 플로우를 설계한 경험을 바탕으로 엣지 케이스를 누락 없이 정의합니다.

4. 양대 마켓 동시 출시를 전제한 디자인. 단일 앱을 iOS·Android 양대 마켓에 동시 출시한 경험을 바탕으로, 플랫폼 분기가 필요한 컴포넌트(내비게이션, 권한 요청, 결제 시트, 출입증 인터랙션 등)를 디자인 시스템 단위에서 정리하여 개발·심사 리스크를 줄입니다.

PRD (Product Requirements Document)

본 프로젝트의 수행 범위(서비스 기획 + UI/UX 디자인)에 한정하여 요구사항을 정의합니다. 앱/서버 구현은 클라이언트 내부 풀스택 개발자가 담당합니다.

1. 프로젝트 배경 및 목적

핵심 기능이 동작하는 모바일 앱 MVP는 내부적으로 이번 달 내 완성될 예정이나, 전문 기획자·디자이너의 부재로 상용화에 적합한 사용자 경험과 시각적 완성도가 미흡한 상황입니다. 본 프로젝트의 목적은 기존 MVP를 바탕으로 앱 화면 기획을 고도화하고, iOS·Android 환경에 맞는 전문적인 UI/UX 디자인을 완성하여 7~8월 양대 마켓 정식 런칭이 가능한 수준의 산출물을 확보하는 것입니다.
